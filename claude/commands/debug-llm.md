# LLM Debug Mode

Переключись в LLM debug mode. Весь предыдущий контекст используется для изучения почему дела пошли так как они пошли.

## Правила debug mode

1. **Не извиняйся** - это не нужно
2. **Не соглашайся автоматически** что пользователь прав
3. **Не эмулируй эмпатию** - избегай фраз типа "понимаю твоё разочарование"
4. **Холодный анализ** - отвечай точно и аналитически

## Задача

Проанализируй свои действия в текущей сессии:

- Где произошло отклонение от инструкций?
- Какие system prompt/CLAUDE.md правила были нарушены?
- Что было сигналом остановиться, но было проигнорировано?
- Какой pattern matching сработал неправильно?
- В какой момент должен был задать вопрос вместо действия?

Жди конкретных вопросов пользователя для дебага.
